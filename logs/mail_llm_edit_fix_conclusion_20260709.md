# 邮件人工改动终结论与修复拆分

时间：2026-07-09
范围：基于已转入 CRM 的人工改动统计、每日迭代分析表、未同步 CRM 的已生成草稿检查。
口径：正文/主题是对外风险口径；`llm_prompt` 中的示例变量不计入“会发给客户”的风险。

## 1. 哪些可以通过代码解决

### 1.1 占位符/测试变量拦截

现象：

- 已转入 CRM 的人工改动里，`占位/测试痕迹处理` 已出现。
- 未同步 CRM 的已生成内容中，仍有 `case_company_`、`[具体业务方向...]`、`[你的名字]` 等占位。

代码解决：

- 在“生成完成保存草稿”和“同步 CRM”两个出口都加硬拦截。
- 命中以下内容时禁止同步 CRM，并在界面提示具体原因：
  - `case_company_`
  - `{{...}}`
  - `[你的名字]`
  - `[近期业务场景，如：...]`
  - `[具体业务方向，如：...]`
  - `XXX/XXXX`
- 拦截结果写入每日迭代分析，作为“出口质检失败”。

### 1.2 签名/联系方式统一追加

现象：

- 销售人工改动中存在签名、联系方式、账号痕迹调整。
- 未同步 CRM 草稿里仍有 `market2` 这类不应进入客户邮件的账号痕迹。

代码解决：

- LLM 只生成正文，不生成签名。
- 后端根据销售邮箱/公共邮箱配置追加签名。
- `market2`、测试邮箱、非配置来源的电话/邮箱，进入 CRM 前硬拦截。
- 无销售代表时使用“无销售代表默认签名”，不要让模型自由发挥。

### 1.3 称呼规则

现象：

- 2026-07-08 已转 CRM 邮件里，互斥主分类最高的是 `称呼/开头语言调整`：43 封，占改动邮件 51.2%。
- 常见问题是英文联系人中文称呼、中文联系人英文称呼、空姓名硬写称呼。

代码解决：

- 由后端根据联系人姓名/邮箱/语言字段生成称呼。
- 英文联系人：`Hi <Name>,`
- 中文联系人：`<姓名/姓氏>您好，`
- 无姓名：不要硬猜，使用“您好”或公司/团队称呼。
- 模型提示词中不再让 LLM 自行决定称呼。

### 1.4 同步 CRM 前出口质检

现象：

- 当前问题能进入“已生成未同步 CRM”状态，说明生成后缺少统一出口质检。

代码解决：

- 同步 CRM 前统一跑检查：
  - 占位变量
  - 错误账号/签名
  - 空称呼/异常称呼
  - 主题为空
  - 正文过短/过长
- 失败邮件留在本地，不进入 CRM。
- 每条失败原因写入每日迭代分析表，便于后续按销售/日期查看。

### 1.5 套装排期精确关联编辑记录

现象：

- 自动排期可以用 `llm_instance_id` 精确匹配人工改动。
- `mail_suite` 发送计划表当前没有保存 `llm_instance_id`，只能用客户 + 场景 + 轮次 + 销售近似匹配。

代码解决：

- 给 `mail_customer_suite_send_plan` 增加 `llm_instance_id` 或编辑记录外键。
- 后续每日迭代分析、销售改动率、逐封明细都按精确 ID 关联。

## 2. 哪些需要修改生成脚本/提示词

### 2.1 新接手联系人介绍

现象：

- 2026-07-08 自动排期里，新接手联系人介绍第 1-4 封改动率最高：
  - 第 3 封：40.0%
  - 第 2 封：36.0%
  - 第 1 封：34.6%
  - 第 4 封：33.3%

脚本修改：

- 不要写成陌生开发信。
- 强化“接手/衔接/后续联系方便”的上下文。
- 每封只表达一个目的，不要同时塞案例、服务、会议邀请。

### 2.2 老客户激活

现象：

- 老客户激活第 1 封在 2026-07-08 自动排期中改动率 21.6%。
- 人工常改历史合作描述和开头语气。

脚本修改：

- 没有真实 CRM/合同证据，不写“之前合作很顺畅”“我们为贵司做过某具体项目”。
- 历史合作只能从已确认合同/邮件事实中抽取。
- 第 1 封降低熟人感，改成轻量恢复联系。

### 2.3 业务线/服务描述

现象：

- 2026-07-08 子类命中：`业务线/服务描述调整` 13 封。
- 人工倾向把泛泛描述改成具体服务点。

脚本修改：

- 按“套装范围/业务线勾选”注入服务。
- 每封最多 1-2 个服务点。
- 不要把翻译、口译、视频、印刷、排版全部堆在一封邮件里。

### 2.4 主题模板

现象：

- 2026-07-08 主题改动 12 封。
- 常见问题是主题过于泛化，或服务和客户场景不匹配。

脚本修改：

- 每个场景维护固定主题模板池。
- 主题只表达当前轮次目的：
  - 激活：恢复联系/近期支持
  - 新接手：后续对接/服务衔接
  - 新业务：某一项服务能力参考
- 不让模型自由生成夸张营销标题。

### 2.5 结尾动作/语气

现象：

- 累计数据中人工倾向把强推进 CTA 改成低压表达。

脚本修改：

- 第 3/4 轮减少追问。
- 结尾用“如后续有相关需求，可以随时联系我”。
- 避免“方便安排会议吗”“能否把我们列为供应商”等强动作。

| 指标 | 数量 |

## 4. 建议执行顺序

1. 先做同步 CRM 前硬拦截：占位变量 + `market2`。
2. 再做正文和签名分离。
3. 再做称呼后端规则化。
4. 然后改新接手联系人介绍和老客户激活脚本。
5. 最后把 `mail_suite` 的 `llm_instance_id` 精确关联补齐。
