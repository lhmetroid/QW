# A 类·动作模板切片草稿（待人工审核后入库）

> 来源：从 V17 90+ 高分回复提炼的通用动作话术，不含真实案例/价格/编造内容。
> 格式遵循《优微记录入知识库标准定义》14.5。审核通过后按 knowledge_chunk 入库（business_line/scenario 字段入库时补）。

## 1. 邮件/附件未收到 → 换方式重发+截图+查垃圾箱
- slice_title：客户没收到邮件或附件时，换方式重发并截图确认
- slice_question（触发语义）：客户说 没收到 / 邮箱没有 / 再发一下 / 附件打不开
- slice_answer（≤50字，可直发）：我换个邮箱再发一遍，发完截图给您，您也看下垃圾箱哈。
- normalized_content：适用于已发邮件/报价/资料但客户未收到。动作=换发送方式（换邮箱/转微信/压缩/转PDF）+发完截图同步+提醒查垃圾箱，至少做其中 2 项。
- usage_boundary：不适用于首次报价、正式合同/文件确认；真实邮箱由销售当场填，禁占位符。
- slice_quality_score：92
- 来源：V17 S04-1 / S07-2 等多条 90+ 回复提炼

## 2. 客户要邮箱/拉线上会 → 安全应答（禁占位符）
- slice_title：客户要销售邮箱以拉线上会时的安全应答
- slice_question：客户说 邮箱发我 / 拉你进会 / 把邮箱给我
- slice_answer：好的，我把邮箱发您微信，您直接拉我进会就行，到点我准时上线。
- normalized_content：客户索要邮箱或拉会时，用「我把账号发您」安全表述+确认到场时间；真实邮箱由销售当场填入。
- usage_boundary：严禁输出占位符 xxx@xxx.com 等；无真实邮箱时只说「我把账号发您」，不编造。
- slice_quality_score：90
- 来源：修正 V17 S10-1 占位符红线对应的正确动作

## 3. 客户压价/要折扣/不开票降价 → 守价口径（申请+反馈时间）
- slice_title：客户压价或要折扣、不开票降价时的守价应答
- slice_question：客户说 能便宜点吗 / 降6% / 不开票少点 / 这价超预算了
- slice_answer：这个我去帮您申请一下，明天上班前给您准信儿。
- normalized_content：涉及降价/折扣/方式票/不开票一律落到「申请或找财务确认 + 明确反馈时间」，不主动暗示可降、不报让步数字。
- usage_boundary：禁止 能降一些/降X%/少X元/给你个低价/算个具体数 等软硬让步；不替财务或领导承诺结果。
- slice_quality_score：92
- 来源：V17 S01-2 / S07-3 守价正例（不答应少6%、转财务确认）提炼

## 4. 巴西市场葡语翻译 → 推荐巴葡（一句说清，不长篇）
- slice_title：客户面向巴西市场做葡语翻译时推荐巴葡
- slice_question：客户说 巴西销售用 / 葡语用哪种 / 没听过这分类
- slice_answer：巴西市场用巴葡最对口，跟欧洲葡语用词有差异，我按巴葡给您准备。
- normalized_content：面向巴西市场的葡语翻译统一推荐巴西葡萄牙语（巴葡）；一句说清差异并顺势定版本，不长篇科普。
- usage_boundary：仅限语种版本选择建议，不涉及报价/交付承诺。
- slice_quality_score：90
- 来源：V17 S10-2 / S03-1 巴葡确认高分回复提炼
