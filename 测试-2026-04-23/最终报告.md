## 发现的问题
- `8000` 端口无法绑定，Windows 返回 `WinError 10013`；本轮改用 `8010`。
- `/api/sync/today` 原本直接在 FastAPI 事件循环内同步调用企微 C-SDK，SDK 长时间未返回时会阻塞页面后续接口。
- 右侧链路按钮 `onclick` 属性双引号嵌套错误，真实点击不会触发 LLM 阶段请求。
- 前端触发类操作原先缺少稳定可见的请求地址、后端返回、失败原因与下一步建议。
- 后端阶段触发原先缺少统一的结构化可追踪日志。

## 已修复内容
- `/api/sync/today` 改为 `asyncio.to_thread()` 执行真实 SDK 调用，并增加 60 秒超时与结构化结果日志，避免阻塞 FastAPI 事件循环。
- 右侧链路按钮统一修复 `onclick` 引号，真实点击可发起请求。
- 前端新增 `operation-status` 状态框，显示已发起、请求地址、后端返回、失败原因和下一步建议。
- 后端新增 `REPLY_CHAIN_*` 日志，覆盖同步请求、阶段触发、步骤开始、步骤成功/失败原因。

## 未解决问题及原因
- 企业微信应用推送卡片返回 `errcode=301002 not allow operate another agent with this accesstoken`。这是企微应用 access token 与 agent 权限配置不匹配，属于外部企微应用配置问题；不影响本次网页端回复生成链路完成。
- `8000` 端口被当前 Windows 环境拒绝绑定，原因是系统端口策略/权限限制；本轮已用 `8010` 完成真实服务验证。
