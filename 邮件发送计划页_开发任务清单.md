### B7. 已发部分套装的处理策略（新增需求）
> 场景：联系人匹配到的套装已发送过其中部分封时，如何继续。支持 4 种模式。
- [ ] B7.1 模式定义：
  - `remaining` 发送部分 = 只补发未发过的封（跳过已发的 step）。
  - `resend_all` 全部重发 = 整套从第1封重新排期生成。
  - `skip` 跳过套装 = 该联系人此套装不发（计入 skipped，可继续匹配下一规则/兜底）。
  - `use_global` 走统一配置 = 用全局默认模式（见 B7.4）。
- [ ] B7.2 规则结构扩展：`mail_autosend_suite_rule.conditions` 同级加字段 `partial_sent_mode`（默认 `use_global`）；前端规则行加下拉选择，浮动说明。
- [ ] B7.3 排期算法扩展：`_autosend_pick_suite`(19981)/`_autosend_schedule`(19998) 读取该联系人该套装"已发到第几封"（来源 `mail_contact_suite_history` / `mail_autosend_plan_item(status=sent)` / `spQueueSend(宣传邮件-AI)`），按模式决定要排哪些 step：
  - remaining → 仅未发 step，周期接已发最后一封时间顺延；
  - resend_all → 全部 step；
  - skip → 不排，记 skipped(reason=partial_sent_skip)；
  - use_global → 取全局默认。
- [ ] B7.4 全局默认：`mail_autosend_config` 加字段 `default_partial_sent_mode`；前端"③ 周期/每日上限"区加全局下拉。
- [ ] B7.5 预览/详情标注：排期清单里每封标记"已发/补发/重发/新发"，让销售一眼看清；节点3 详情同样标注。
