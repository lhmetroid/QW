# 全部功能和使用说明-智能销售辅助工作台-20260427

本文档是当前项目的主说明文档。以后凡是界面按钮、选项卡、菜单、表头、字段名、导入模板、审核流程、训练流程有改动，必须同步更新本文件。

## 1. 项目定位

智能销售辅助工作台用于两类工作：

- 企微实时智能：读取真实会话、抽取意图、合并 CRM 与线程业务事实、检索知识证据、生成销售建议、收集反馈。
- 知识库管理：导入知识、审核发布、治理报价规则、质量巡检、回归测试、查看命中日志、执行补齐与训练准备。

## 2. 进入方式

- 启动后端：`powershell -NoProfile -ExecutionPolicy Bypass -File .\start_backend.ps1`
- 访问地址：`http://127.0.0.1:8000/`
- 顶部双入口：
- `企微实时智能`
- `知识库管理`

## 3. 界面通用说明

- 现在所有主要按钮、选项卡、菜单、表格表头、字段名、表单输入项都带浮动使用说明。
- 鼠标悬停即可看到说明；键盘聚焦时也会显示。
- 说明覆盖按钮动作、筛选条件、字段含义、表头解释、风险提示、导入规范。

## 4. 企微实时智能功能

- `同步今日真实记录`：同步当天真实企微会话，不写模拟数据。
- 左侧会话列表：查看真实会话，点击后进入当前线程分析。
- `刷新分析`：刷新摘要、画像、线程事实、知识命中与建议结果。
- `立刻调用 AI 提取特征` / `重新提取特征`：重新生成结构化摘要与业务特征。
- `立刻调用知识库进行话术辅助生成` / `重新演算金牌建议`：基于已发布知识、CRM、线程事实生成建议。
- `复制`：复制建议内容。
- `有用`：将当前建议记为正向反馈，并回流效果分。
- `需修正`：登记修正意见，并自动转候选知识。
- `引擎配置`：维护规则脚本、第一阶段特征提取提示词、第二阶段回复生成提示词。
- 右侧链路面板：查看会话概况、线程业务事实层、客户画像摘要、最新消息、命中证据、生成链路与操作状态。

## 5. 知识库管理功能

### 5.1 选项卡

- `列表`：按文档阶段、知识分类、服务筛选全部知识文档。
- `待审核`：批量提交审核、批量审核同意、批量发布、批量归档。
- `知识候选`：查看销售反馈、会话抽取、邮件整理表、优秀回复抽取生成的候选知识。
- `导入中心`：下载模板、预览 Excel、确认导入、执行历史抽取、查看异步导入任务。
- `批次`：查看每次导入批次，支持审核、发布、归档、回滚。
- `报价规则`：查看结构化报价规则。
- `检索测试`：手工输入问题，验证命中、分数、过滤与排序。
- `质量治理`：查看缺报价、冲突、过期、混合知识等问题并修复。
- `回归测试`：按分组、类别、风险、服务执行异步回归。
- `运行状态`：检查数据库、Embedding、LLM、CRM、企微、会话存档组件配置。
- `命中日志`：查看真实检索命中日志，并转候选知识。

### 5.2 文档与切片

- `新增知识`：手工创建草稿知识。
- `详情`：查看文档、切片、报价规则、版本历史、发布门禁结果。
- `编辑文档`：修改文档基础字段。
- `编辑知识切片`：修改拆分后的知识片段。
- `补报价规则` / `补规则`：为相关切片补结构化规则。
- `保存为草稿`：进入草稿，不参与正式检索。
- `提交审核`：提交到审核流程。
- `审核同意`：进入可发布状态。
- `发布` / `执行发布上线`：通过门禁后进入正式检索面。
- `归档`：停止参与检索但保留历史。
- `驳回`：审核不通过。
- `恢复到此版本`：按历史快照恢复。

### 5.3 候选知识

- 候选知识不能直接参与检索，也不能直接发布。
- `编辑候选`：修正文案、分类、字段。
- `转为草稿知识`：进入正式知识流程。
- 候选来源当前包括：
- `feedback`
- `message_extract`
- `case_extract`
- `email_excel`
- `excellent_reply`

### 5.4 导入中心

- `下载模板`：下载当前统一导入模板。
- `预览`：只校验，不写库。
- `确认导入`：将通过校验的数据写入草稿文档。
- `会话抽取`：从企微会话沉淀候选知识。
- `历史案例抽取`：从人工整理的案例文本沉淀候选知识。
- `邮件整理表抽取`：从邮件整理 Excel 沉淀候选知识。
- `异步导入任务`：查看后台导入任务状态。

### 5.5 检索、质量、回归

- `检索测试`：验证特定问题在当前条件下的命中、排序、证据和报价规则。
- `质量治理`：聚合当前可修复问题并提供跳转操作。
- `回归测试`：验证知识发布后对关键业务问题是否仍能正确命中。
- `命中日志`：复盘真实查询、未命中原因、耗时，并可转候选。

## 6. 导入 Excel 模板说明

### 6.1 当前统一模板支持的内容

- FAQ常见问答
- 流程规则
- 能力知识
- 案例
- 邮件模板
- 名词定义
- 报价限制条件
- 结构化报价规则

### 6.2 当前模板字段

- `标题`
- `内容`
- `知识分类`
- `切分`
- `服务`
- `语种`
- `服务范围`
- `客户层级`
- `优先级`
- `风险等级`
- `生效日期`
- `失效日期`
- `单位`
- `币种`
- `价格`
- `最高价格`
- `最低收费`
- `加急倍率`
- `税费`
- `标签`

### 6.3 填写规则

- 普通知识至少填写 `标题`、`内容`，并建议填写 `知识分类`。
- 结构化报价规则行保持 `知识分类` 留空，同时填写 `价格`、`单位`、`币种`、`生效日期`、`失效日期`；有区间时补 `最高价格`，有起步价时补 `最低收费`。
- `切分=1` 表示允许系统按业务知识点继续拆分。
- `风险等级` 推荐按 `低 / 中 / 高` 填写。
- `标签` 使用逗号分隔。

### 6.4 模板与样例文件

- 下载模板接口：`/api/kb/import/template/download`
- 模板工作簿现已内置：
- 主数据页
- `填写说明`
- `字段说明`
- `枚举参考`
- 主数据页已附完整样例行，可直接参照填写。
- 项目内同步更新的样例文件：`docs/导入中心Excel完整示例.xlsx`

## 7. 后台补齐与训练准备功能

- `POST /api/kb/complete_partial_items`：补齐历史“部分完成项”，实际回填线程事实、邮件主表、邮件片段、效果反馈、训练样本。
- `POST /api/ml/run_training_pipeline`：执行训练样本准备、导出与 runner 编排。
- `backend/complete_all_analysis_items.py`：一键补齐脚本，适用于当前库历史数据回填。
- 正向反馈会自动触发优秀回复抽取与训练样本增量准备。

## 8. 改造后与原有方案的区别点

- 原方案更偏“知识文档检索 + 回复生成”；现方案增加了线程级业务事实层，不再只靠文档和摘要判断。
- 原方案事实与参考知识未彻底分层；现方案已形成 `ThreadBusinessFact` 事实层与知识文档/切片参考层双数据面。
- 原方案邮件更多作为原始来源；现方案已形成邮件主表、邮件片段表、邮件效果反馈表，并能沉淀优秀回复片段。
- 原方案附件信息主要是弱引用；现方案会做附件结构化汇总并合入线程事实。
- 原方案导入模板偏单页字段提示；现方案模板下载即附带填写说明、字段说明、枚举参考和完整样例。
- 原方案界面对新用户依赖口头培训；现方案在按钮、选项卡、菜单、表头、字段名上统一增加浮动使用说明。
- 原方案训练准备停留在思路层；现方案已具备训练样本准备、导出、runner 编排入口。
- 原方案反馈回流较弱；现方案已把正负反馈、effect_score、feedback_count 接回事实层、切片层、邮件资产层与训练样本层。
- 原方案更依赖单次检索相似度；现方案排序显式引入质量分、效果分、生成分、意图分和可回复性。
- 原方案发布门禁主要校验基础结构；现方案进一步校验混合知识点、可生成许可、可回复性、线程状态一致性。

## 9. 新用户推荐操作顺序

1. 先在 `知识库管理 -> 导入中心` 下载模板并看模板内的说明页与样例行。
2. 先用 `预览` 修正 Excel，再执行 `确认导入`。
3. 到 `待审核` 完成审核、发布。
4. 用 `检索测试` 验证关键问题命中结果。
5. 用 `质量治理` 和 `回归测试` 做上线前检查。
6. 企微使用中如发现建议有问题，用 `需修正` 或命中日志中的 `转候选` 继续回流。
