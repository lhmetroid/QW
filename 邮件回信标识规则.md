# 邮件回信识别规则 (发送端 ↔ 读取端约定)

> 目的: 让"邮件统计"能把客户回信精确对应到我方发出的那一封 AI 邮件。
> 分工: **发送端按本规则写 Message-ID**; **读取端(本系统 mail_ai_stats)只解析回信、按规则提取标识匹配**。

## 背景(为什么必须用 Message-ID)

- 客户回信时, 邮件客户端会在回信头里**标准回带原邮件的 `Message-ID`**, 放进 `In-Reply-To` 和 `References`。
- 自定义头(如 `X-Speed-AI-Id`)**不会被回信带回来**, 所以标识只能放在 `Message-ID` 里。
- CRM 库里没有 `In-Reply-To` 列, 但收信原文 `.eml` 存在 FTP(`spReceiveInfo{销售}.EmlFtpDir`), 头部完整, 可解析。

## 发送端规则(发送端实现)

发送每封 AI 邮件时, 把邮件头 `Message-ID` 设为内嵌 `SendId` 的确定值:

```
Message-ID: <AIMAIL-{SendId}-{uniq}@speed-asia.com>
```

| 占位 | 含义 |
|---|---|
| 固定前缀 `AIMAIL-` | 读取端识别用, 不可改 |
| `{SendId}` | spQueueSend 触发器生成的 SendId, 形如 `Mal_S260622-000003`, 发送端本就持有 |
| `{uniq}` | 任意全局唯一随机串(GUID/时间戳), 保证 Message-ID 不重复 |
| 域名 | 用我方发件域 `speed-asia.com` |

示例:
```
Message-ID: <AIMAIL-Mal_S260622-000003-7f3a9c2b@speed-asia.com>
```

要点:
1. 只有 UseRange=`宣传邮箱-AI`(本系统转入的 AI 邮件)才按此规则写, 其他邮件不受影响。
2. `SendId` 原样放入, 不要改写其中的下划线/连字符。
3. 一封邮件一个唯一 Message-ID。

## 读取端(本系统, 已实现)

- 客户回信落 `spReceiveInfo{销售}.EmlFtpDir`, 拉 `.eml` 解析 `In-Reply-To` + `References`。
- 正则提取 SendId: `AIMAIL-(Mal_S\d{6}-\d{6})` (见 `backend/mail_ai_stats.py: extract_send_ids_from_refs`)。
- 用提取到的 SendId 匹配本地 `mail_customer_suite_send_plan.crm_send_id` → 该回信归属到对应的那封 AI 邮件, `match_method='eml_message_id_rule'`。
- 兜底: 发送端规则未上线前, 读取端会去拉发送 `.eml` 解析系统自生成的 `Message-ID` 再匹配(`match_method='eml_in_reply_to'`), 规则上线后可不依赖此兜底。

## 联调验证步骤

1. 发送端按规则发一封 AI 邮件, 在发送 `.eml` 里确认 `Message-ID` 形如 `<AIMAIL-Mal_S...-...@speed-asia.com>`。
2. 用该客户邮箱回一封信。
3. 在"邮件统计"页点"从CRM刷新", 回信数 +1; 双击该天回信数看明细, 确认对应到正确的客户/主题, `match_method=eml_message_id_rule`。
