# 软件使用说明和技术介绍

本文档聚焦当前系统的知识库部分，说明已有功能、推荐操作顺序、关键按钮用途、技术实现方法和发布门禁规则。

## 1. 入口与页面结构

启动后台后，在浏览器打开：

```text
http://127.0.0.1:8000/
```

页面顶部有两个并排主节点：

- `企微实时智能`：用于查看企微真实会话、提取客户意图、读取客户画像、生成销售建议。
- `知识库管理`：用于知识导入、查看、审核、发布、检索测试、质量治理、回归测试和命中日志分析。

进入知识库：

1. 点击顶部 `知识库管理`。
2. 在知识库页签中选择功能。
3. 常用直达地址：`http://127.0.0.1:8000/#kb-import` 可直接进入导入中心。

知识库页签包括：

- `列表`
- `待审核`
- `知识候选`
- `导入中心`
- `批次`
- `报价规则`
- `检索测试`
- `质量治理`
- `回归测试`
- `运行状态`
- `命中日志`

页面节点与实际用途：

- `列表`：知识文档总列表。页面顶部下拉筛选包括 `知识状态`、`知识类型`、`业务线`。其中知识状态为 `草稿`、`审核中`、`审核同意`、`已发布`、`已归档`、`已驳回`。
- `待审核`：集中处理 `草稿`、`审核中`、`审核同意` 三类知识，方便批量 `提交审核`、`审核同意`、`发布`、`归档`。
- `知识候选`：候选池，不是正式知识库。用于承接销售反馈、会话抽取、历史案例抽取和邮件整理表抽取出来的待确认内容。
- `导入中心`：下载模板、选择文件、预览校验、确认导入，并生成异步导入任务。
- `批次`：按导入批次编号查看导入结果，并对同一批知识批量提交审核、审核同意、发布、归档或回滚。
- `报价规则`：查看已结构化的价格规则，例如语种、计价单位、价格、最低收费、有效期。
- `检索测试`：手工输入客户问题，验证已发布知识是否能真实命中。
- `质量治理`：发现缺报价规则、报价冲突、重复规则、过期知识等问题。
- `回归测试`：按标准验收用例批量验证知识库检索质量和发布门禁规则。
- `运行状态`：检查数据库、向量模型、大模型、客户画像库、企微组件等配置和连通状态。
- `命中日志`：查看真实检索记录，并可把问题反馈转为知识候选。

## 2. 核心概念

### 2.1 知识类型

`能力知识`：
说明公司能不能做、支持哪些语种、支持哪些服务范围。适合回答“中译德能做吗”“是否支持德语翻译”“业务范围有哪些”。

`报价知识`：
说明价格、计价单位、最低收费、税费、折扣、加急费。报价知识属于高风险知识，必须尽量配套结构化报价规则。发布时门禁会重点检查。

`流程知识`：
说明下单、付款、审核、交付、售后、开票等操作步骤。适合回答“怎么下单”“付款流程是什么”“交付怎么走”。

`常见问答`：
承载普通解释、问答、话术和案例摘要。常见问答不能替代结构化报价规则，也不能直接承诺未结构化的价格。

### 2.2 切片类型

`规则`：
可执行、可判断的标准条款，例如报价规则、付款规则、服务边界。规则适合作为高可信检索证据。

`问答`：
一问一答式知识，用于稳定、低风险的标准回答。

`案例`：
历史成交或沟通案例，用于参考场景和话术，不能直接当作当前承诺。

`话术模板`：
销售可复用表达模板。模板必须基于已发布知识证据使用，不能编造价格、交期或能力。

`限制条件`：
不能做、需要人工确认、需要补充信息等边界条件。命中限制条件时通常要求人工复核。

`定义`：
解释术语、字段和业务概念，用于提升检索理解，不承载业务承诺。

### 2.3 知识文档状态

`草稿`：
已写入数据库，但未审核，不参与正式检索。

`审核中`：
已提交审核，等待人工审核结论，尚未参与正式检索。

`审核同意`：
审核已经通过，但还没有执行发布。只有 `审核同意` 状态的数据才能点击 `发布`，并进入发布门禁检查。

`已发布`：
已通过发布门禁并正式上线，参与检索和销售建议生成。

说明：`已发布` 不等于单纯“审核通过”。系统现在按单一主状态展示，真实顺序是 `草稿 -> 审核中 -> 审核同意 -> 已发布`。

`已归档`：
停止参与检索，保留历史记录。

`已驳回`：
审核不通过，不能发布。

### 2.4 知识候选状态

知识候选是“候选池”阶段，不是正式知识文档，也不是草稿。候选不会参与正式检索，必须点击 `转为草稿知识` 后才进入知识文档流程。

`候选中`：
还在候选池，等待人工查看、编辑、决定是否入库。

`已转为草稿知识`：
候选已经提升为知识文档，状态进入 `草稿`，后续按 `提交审核`、`审核同意`、`发布` 流程处理。

`已拒绝`：
候选被人工判定不入库，不参与检索。

`已归档`：
候选作为历史记录保留，不再作为待处理项。

### 2.5 知识候选和草稿有什么不同

这两个阶段名字接近，但在系统里是两个完全不同的池子。

`知识候选`：

- 所在页面：`知识候选`
- 所属池子：候选池
- 是否已经是正式知识文档：不是
- 是否已经生成正式知识切片和发布版本：不是正式发布对象
- 是否参与正式检索：不参与
- 是否能直接发布：不能
- 主要用途：先承接历史抽取、销售反馈、邮件抽取等“待人工判断是否值得入库”的内容
- 后续动作：`编辑候选` 或 `转为草稿知识`

`草稿`：

- 所在页面：`列表`、`待审核`
- 所属池子：正式知识文档池
- 是否已经是正式知识文档：是
- 是否已经写入知识文档/知识切片/结构化字段：是
- 是否参与正式检索：不参与
- 是否能直接发布：不能，必须继续走 `提交审核 -> 审核同意 -> 发布`
- 主要用途：作为正式知识入库后的审核前阶段
- 后续动作：`提交审核`、`审核同意`、`发布`

一句话区分：

- `知识候选` 是“还没决定是否正式入库”的待筛选内容。
- `草稿` 是“已经决定入库，并已经写成正式知识文档”的待审核内容。

### 2.5.1 这是不是多此一举

不是同一件事做两遍，而是两道不同性质的关口。

`知识候选` 这一关解决的是：

- 这段内容值不值得入库
- 抽取结果有没有偏差
- 文本是否完整、是否需要人工改写
- 是否只是临时反馈，不适合沉淀为正式知识

`草稿` 这一关解决的是：

- 已决定入库后，如何作为正式知识文档落库
- 是否补齐知识类型、切片类型、业务线、服务范围等结构化字段
- 报价类是否补齐结构化报价规则
- 是否通过审核、发布前结构校验和发布门禁

因此：

- 对 `会话抽取`、`历史案例抽取`、`邮件整理表抽取`、`销售建议需修正反馈` 这类高噪声来源，`知识候选 -> 草稿` 有必要。
- 对已经人工整理好的正式资料，例如 `报价规则`、`流程规范`、`能力清单`、`常见问答` Excel 导入，直接进入 `草稿` 即可，不需要先进候选。

当前系统也正是这样分流的：

- `正式导入 / 手工新增`：直接进 `草稿`
- `历史抽取 / 邮件抽取 / 销售反馈`：先进 `知识候选`

所以系统并不是要求所有数据都走两遍，而是按数据质量和来源做分流处理。

### 2.6 为什么知识类型和切片类型要分成两个维度

有必要分成两个维度，因为它们解决的是两件不同的事。

`知识类型` 决定的是业务含义：

- 这条知识属于能力、报价、流程，还是常见问答。
- 检索时应该匹配哪类客户问题。
- 发布门禁和风险规则应按哪类知识检查。

`切片类型` 决定的是证据形态：

- 这条知识是规则、问答、案例、话术模板、限制条件还是定义。
- 命中后应该如何解释和使用它。
- 它在证据优先级和人工复核中扮演什么角色。

如果只保留一个维度，会出现表达不清的问题：

- 同样是 `报价知识`，可能是 `规则`，也可能是 `限制条件`。
- 同样是 `能力知识`，可能是 `规则`，也可能是 `案例`。
- 同样是 `常见问答`，可能是 `问答`，也可能是 `话术模板` 或 `定义`。

可以把它理解为：

- `知识类型` = 这条知识在业务上属于什么问题
- `切片类型` = 这条知识以什么证据形式存在

推荐判断顺序：

1. 先判断“它回答的是哪类业务问题” -> 选 `知识类型`
2. 再判断“它以什么形式最安全、最清楚” -> 选 `切片类型`

典型组合示例：

- `能力知识 + 规则`：支持哪些语种、哪些服务范围可以做
- `能力知识 + 案例`：过去做过哪些相似项目
- `报价知识 + 规则`：价格、单位、最低收费、税费
- `报价知识 + 限制条件`：某些范围必须人工复核或特殊报价
- `流程知识 + 规则`：标准步骤、节点、交付要求
- `流程知识 + 问答`：流程类常见解释
- `常见问答 + 问答`：普通客户问答
- `常见问答 + 话术模板`：销售表达模板
- `常见问答 + 定义`：术语解释

对当前系统来说，这两个维度都参与：

- 检索过滤
- 门禁校验
- 人工复核判断
- 前端展示说明

所以不建议把 4 类知识类型和 6 类切片类型合并成一个维度。

## 3. 标准入库流程

推荐顺序：

1. 进入顶部 `知识库管理`。
2. 点击 `导入中心`。
3. 在 `导入类型` 选择数据类别。
4. 点击 `下载模板`。
5. 按模板填写表格。
6. 回到 `导入中心`，点击 `选择文件`。
7. 点击 `预览`。
8. 如果预览显示失败行，先修正表格。
9. 预览无阻断错误后，点击 `确认导入`。
10. 导入完成后系统会生成导入批次编号。
11. 点击 `批次`。
12. 选择刚生成的批次。
13. 点击 `提交审核`。
14. 审核人员点击 `审核同意`。
15. 点击 `发布`。
16. 发布门禁通过后，知识进入 `已发布`，正式参与检索。

### 3.1 导入类型怎么选

`常见问答`：
适合一般问答、服务说明、客户常问问题。

`报价规则`：
适合报价单、价格表、计价标准。应填写价格、计价单位、币种、生效日期和失效日期。

`流程规范`：
适合业务流程、付款流程、审核流程、交付流程。

`能力清单`：
适合服务范围、语种能力、行业覆盖、可承接内容。

`历史案例`：
适合成交案例、沟通案例、客户异议处理案例。

`销售话术`：
适合可复用表达模板，但必须注意不能替代报价和流程规则。

`邮件整理`：
适合从整理后的历史邮件中抽取候选知识。

### 3.1.1 这 4 类知识导入后分别先进哪里

如果是通过 `导入中心` 的 Excel 导入：

- `常见问答`
- `报价规则`
- `流程规范`
- `能力清单`

这 4 类都会直接进入 `草稿`，不会先进 `知识候选`。

系统实际动作是：

- 先进入导入任务。
- 导入任务完成后写入正式知识文档和知识切片。
- 生成 `导入批次编号`。
- 文档主状态为 `草稿`。
- 后续到 `批次` 或 `待审核` 继续 `提交审核 -> 审核同意 -> 发布`。

其中：

- `报价规则` 导入时，如果表格里有价格、单位、币种等结构化字段，会同时生成结构化报价规则。
- `常见问答`、`流程规范`、`能力清单` 导入后主要生成知识文档和知识切片，一般不会自动生成报价规则，除非内容本身被识别为报价信息。

### 3.1.2 哪些内容会先进入知识候选

以下不是直接导入正式知识，而是先进入 `知识候选`：

- 销售建议点击 `需修正` 产生的反馈候选
- `导入中心` 中的 `会话抽取`
- `导入中心` 中的 `历史案例抽取`
- `导入中心` 中的 `邮件整理表抽取`

这些路径的共同点是：

- 先做历史抽取或反馈整理
- 结果先进 `知识候选`
- 人工确认后点击 `转为草稿知识`
- 转成草稿后再进入正式审核发布流程

所以可以这样记：

- `Excel 导入知识`：直接进 `草稿`
- `历史抽取 / 邮件抽取 / 销售反馈`：先进 `知识候选`

### 3.1.3 按 4 类知识类型看，分别会先进哪里

如果一条内容已经通过人工整理为正式导入数据，或者在 `列表` 中手工新增，那么无论它属于哪一种知识类型，都会直接进入 `草稿`：

- `能力知识`：直接进 `草稿`
- `报价知识`：直接进 `草稿`
- `流程知识`：直接进 `草稿`
- `常见问答`：直接进 `草稿`

区别只在于后续处理不同：

- `报价知识`：通常还需要结构化报价规则，发布门禁最严格
- `能力知识`：重点检查是否有真实能力证据，避免夸大能力
- `流程知识`：重点检查是否有流程证据、是否命中有效步骤说明
- `常见问答`：更适合解释性内容、标准回复、话术和定义

如果一条内容来自“抽取”而不是“正式导入”，即使最终被识别成上述 4 类知识类型之一，也会先进入 `知识候选`，不会直接变成草稿。典型来源包括：

- 会话抽取
- 历史案例抽取
- 邮件整理表抽取
- 销售反馈生成候选

### 3.1.4 导入中心 Excel 是否通用

导入中心的正式知识 Excel 是通用格式，推荐业务人员按“最小模板”填写：只填 `标题` 和 `内容`。

页面上的 `常见问答`、`报价规则`、`流程规范`、`能力清单`、`历史案例`、`销售话术`、`邮件整理` 这 7 个导入类型，底层使用同一套 Excel 列结构。区别在于：

- 选择不同 `导入类型` 时，系统会给空白结构字段套用默认 `知识类型`、`切片类型` 和 `风险等级`。
- `服务` 会优先按标题和内容做基础推断，无法判断时归为 `通用`。
- 报价类内容会尝试从正文中识别价格、最低收费、加急倍率、税费等结构化报价字段。
- 其他结构字段不是业务人员必须填写的字段，应在导入后的草稿审核环节校准。
- 正式知识导入只支持 `.xlsx` / `.xlsm`。导入后直接进入 `草稿`，不先进 `知识候选`。
- `邮件整理表抽取`、`会话抽取`、`历史案例抽取` 属于抽取入口，结果先进 `知识候选`，不是同一条正式 Excel 入库链路。

当前正式 Excel 入库是“一行生成一条草稿知识”。如果一段内容里混有多个独立报价、多个流程节点或多个案例，建议在导入前拆成多行，或先走抽取/辅助整理入口生成候选，再由人工确认后转草稿。

### 3.1.5 通用 Excel 字段说明

当前通用模板共 20 列。

| 字段 | 是否需要人工填写 | 填写方式 | 说明 |
|---|---:|---|---|
| 标题 | 必填 | 文本 | 知识标题、问题、场景名或规则名。 |
| 内容 | 必填 | 文本 | 知识正文、答案、流程步骤、话术、案例或报价内容。 |
| 知识类型 | 可留空 | 固定选项 | 业务含义。空值使用导入类型默认值，导入后可审核校准。 |
| 切片类型 | 可留空 | 固定选项 | 证据形态。空值使用导入类型默认值，导入后可审核校准。 |
| 服务 | 可留空 | 固定选项 | 服务分类。系统会按标题和内容基础推断，无法判断时为 `通用`。 |
| 语种 | 可留空 | 固定选项 | 翻译、译制、口译相关知识可填写；也可导入后审核补齐。 |
| 服务范围 | 可留空 | 固定选项 | 资料类型或适用范围，用于检索过滤和发布门禁。 |
| 客户层级 | 可留空 | 固定选项 | 仅当知识只适用于某类客户时填写。 |
| 优先级 | 可留空 | 数字 | 默认 `50`。审核时可提高关键规则优先级。 |
| 风险等级 | 可留空 | 固定选项 | 报价内容会自动按高风险处理。 |
| 生效日期 | 报价类建议填 | 日期 | 支持 `2026-04-20`、`2026/04/20` 等格式。 |
| 失效日期 | 报价类建议填 | 日期 | 报价类发布前通常需要补齐，否则可能被门禁拦截。 |
| 单位 | 报价类建议填 | 固定选项 | 结构化报价规则的计价单位。 |
| 币种 | 报价类建议填 | 文本 | 默认 `CNY`。 |
| 价格 | 报价类建议填 | 数字 | 最低价或单价，会写入结构化报价规则。 |
| 最高价格 | 可留空 | 数字 | 区间报价的最高价。 |
| 最低收费 | 可留空 | 数字 | 起步价或最低收费。 |
| 加急倍率 | 可留空 | 数字 | 例如 `1.5` 表示 1.5 倍。 |
| 税费 | 可留空 | 文本 | 例如 `不含税`、`含税`、`税点6%`。 |
| 标签 | 可留空 | 文本 | 用于人工识别来源，例如 `报价`、`流程`、`话术`。 |

填写原则：

- 对业务人员，推荐只整理 `标题`、`内容` 两列。
- 对知识库管理员，可在导入前补充固定选项，提高自动归类准确度。
- 固定选项在模板里统一用中文展示；系统也兼容内部系统值。
- 结构字段不是越多越好。没有把握时留空，让导入默认值和草稿审核流程处理。

### 3.2 预览按钮

点击 `预览` 后系统只做解析和校验，不写入数据库。

预览会检查：

- 是否存在标题和内容。
- 知识类型是否合法。
- 业务线是否合法。
- 报价类是否有结构化报价字段。
- 日期格式是否可解析。
- 可导入行、跳过行和失败行数量。

如果看到失败行，应先修正原表格，再重新预览。

### 3.3 确认导入按钮

点击 `确认导入` 后系统会：

- 将合法行写入知识文档表。
- 为每条知识生成知识切片。
- 调用真实向量模型生成向量。
- 报价类会写入结构化报价规则。
- 生成导入批次编号。
- 状态默认为草稿。

导入后不会自动发布，必须走审核和门禁。

## 4. 手工新增知识

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `列表`。
3. 点击 `新增知识`。
4. 填写标题和内容。
5. 选择知识类型、业务线、切片类型、服务范围。
6. 点击 `保存为草稿`。
7. 在右侧详情中检查内容。
8. 点击 `提交审核`。
9. 审核确认后点击 `审核同意`。
10. 点击 `发布`。

注意：

- 手工新增会真实调用向量模型。
- 报价类知识必须补结构化报价规则，否则发布会被拦截。
- 修改知识切片后会重新生成向量。

## 5. 知识候选流程

知识候选来源：

- 销售建议反馈里的 `需修正`。
- 企微会话抽取。
- 历史案例文本抽取。
- 邮件整理表抽取。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `知识候选`。
3. 按候选状态或来源筛选。候选状态包括 `候选中`、`已转为草稿知识`、`已拒绝`、`已归档`。
4. 点击候选记录右侧 `详情`。
5. 如需修正，点击 `编辑候选`。
6. 确认后点击 `转为草稿知识`。
7. 到 `列表` 或 `待审核` 继续执行 `提交审核`、`审核同意`、`发布`。

规则：

- 候选知识不会直接参与检索。
- 候选必须转为草稿知识后，才能提交审核、审核同意和发布。
- `候选中` 表示仍在候选池，不是草稿，也不是审核中。
- `已转为草稿知识` 表示候选已经生成正式知识文档草稿。
- 大模型抽取没有明确知识点时允许返回空候选，不生成兜底问答。

## 6. 批次治理流程

批次适合处理一次导入产生的多条知识。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `批次`。
3. 选择导入批次编号。
4. 查看批次总数、状态分布、文档列表。
5. 点击 `提交审核`，把批次内草稿提交审核。
6. 点击 `审核同意`，把批次内审核中的知识转为可发布状态。
7. 点击 `发布`，对批次内审核同意知识执行发布门禁。
8. 如导入错误，点击 `归档` 或 `回滚`。

按钮说明：

- `提交审核`：把批次知识从草稿变成审核中。
- `审核同意`：审核通过当前批次知识，但不会直接上线。
- `发布`：只允许发布审核同意的数据，执行结构校验和发布门禁，通过后进入已发布。
- `归档`：停止该批次知识参与检索。
- `回滚`：非破坏性回滚，把该批次知识归档并标记为已回滚。

## 7. 报价规则维护

报价规则用于把价格从正文中结构化出来，避免销售建议只凭文本猜价格。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `报价规则`，查看现有规则。
3. 如需为某条知识补报价，进入 `列表`。
4. 点击对应知识的 `详情`。
5. 在报价规则区域点击 `补报价规则`。
6. 填写价格、计价单位、币种等。
7. 如规则错误，点击 `编辑` 或 `停用`。

建议字段：

- 业务线。
- 语种方向。
- 服务范围。
- 计价单位。
- 币种。
- 最低价格。
- 最高价格。
- 最低收费。
- 税费说明。
- 生效时间。
- 失效时间。

报价知识发布要求：

- 必须有有效期。
- 必须有结构化报价规则。
- 高风险报价会触发更多发布门禁用例。

## 8. 检索测试

检索测试用于验证已发布知识是否能被正确命中。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `检索测试`。
3. 输入客户问题。
4. 可选：选择业务线、知识类型、语种、服务范围。
5. 设置返回数量。
6. 点击 `检索`。
7. 查看命中知识、报价规则、置信度、是否需要人工复核。

判断结果：

- 命中报价问题时，应看到结构化报价规则。
- 如果证据不足，系统会标记需要人工复核。
- 如果命中限制条件，也会要求人工复核。
- 检索结果会写入命中日志，便于后续质量治理。

## 9. 质量治理

质量治理用于发现知识库中会影响上线使用的问题。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `质量治理`。
3. 查看问题列表。
4. 点击问题右侧按钮处理：
   - `文档`：打开关联文档。
   - `知识切片`：查看切片内容。
   - `规则`：查看关联报价规则。
   - `补规则`：补结构化报价规则。
   - `归档重复`：归档重复报价规则。
   - `延长期限`：延长即将过期文档。
   - `停用规则`：停用过期或冲突规则。

当前质量检查包括：

- 已发布报价切片缺少结构化报价规则。
- 同一报价范围存在重复规则。
- 同一报价范围存在冲突价格。
- 文档已过期或即将过期。
- 报价规则已过期或即将过期。

## 10. 回归测试

回归测试用于用标准验收问题验证当前知识库。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `回归测试`。
3. 可选：选择分组、类别、风险等级、业务线。
4. 点击 `筛选` 查看用例。
5. 点击 `异步执行回归`。
6. 查看任务结果。

回归测试用途：

- 验证报价问题必须命中报价规则。
- 验证能力问题必须命中能力证据。
- 验证跨业务线不误命中。
- 验证低置信度问题触发人工复核。
- 验证过期、税费、客户层级等风险规则。

## 11. 命中日志

命中日志用于追踪真实检索效果。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `命中日志`。
3. 查看查询文本、命中数量、耗时、未命中原因。
4. 如果某条命中需要补知识，点击 `转候选`。
5. 到 `知识候选` 中审核并转为草稿知识。

## 12. 运行状态

运行状态用于确认真实依赖是否就绪。

操作顺序：

1. 进入 `知识库管理`。
2. 点击 `运行状态`。
3. 查看健康检查、检索性能和配置诊断。

检查项：

- 知识库数据库。
- 向量模型服务。
- 第一阶段大模型。
- 第二阶段大模型。
- 客户画像库。
- 企微应用。
- 会话存档组件。

知识库核心运行至少需要：

- 知识库数据库可连接。
- 向量模型服务可用。
- 第一阶段和第二阶段大模型配置可用。

客户画像和会话存档属于增强功能，不影响知识库导入、检索、审核和发布。

## 13. 技术方法

### 13.1 后端服务

后端采用 Python Web 服务，提供知识库管理接口、导入接口、检索接口、回归接口和质量治理接口。

主要能力：

- 统一读取 `.env` 配置。
- 自动检查并补齐数据库字段。
- 提供知识文档、知识切片、报价规则、候选池、命中日志等接口。
- 提供异步任务机制，用于长耗时导入和回归测试。

### 13.2 数据存储

知识库新数据写入 `AISaleData` 知识库数据库。

核心数据对象：

- `knowledge_document`：知识文档，保存标题、业务线、知识类型、状态、版本、有效期。
- `knowledge_chunk`：知识切片，保存正文、向量、语种、服务范围、客户层级。
- `pricing_rule`：结构化报价规则，保存价格、币种、单位、税费、最低收费、有效期。
- `knowledge_candidate`：候选知识，承接反馈、会话抽取、邮件抽取结果。
- `knowledge_hit_logs`：检索命中日志，记录查询、命中、置信度和人工复核状态。
- `job_tasks`：异步任务记录，用于导入和回归。

客户画像旧库只读使用，不作为知识库存储。

### 13.3 向量化与检索

导入知识时，系统会：

1. 拼接标题和内容。
2. 调用真实向量模型生成向量。
3. 保存向量和结构化标签。
4. 检索时按业务线、知识类型、语种、服务范围、客户层级、有效期过滤。
5. 再结合关键词预筛和向量相似度排序。
6. 返回命中知识、证据引用、结构化报价规则和置信度。

系统不使用假向量。向量模型失败会直接报错。

### 13.4 大模型辅助

第一阶段大模型：

- 从真实会话中提取结构化销售意图。
- 输出客户需求、关键事实、风险、待确认项。

知识库候选抽取：

- 从会话、历史案例、邮件整理表中抽取候选知识。
- 只抽取原文明确存在的内容。
- 不编造价格、能力、流程。

第二阶段大模型：

- 基于已发布知识证据和客户画像生成销售建议。
- 输出会经过证据校验。
- 如果建议中包含未被报价规则支持的价格数字，会被拦截。

### 13.5 严格真实数据策略

系统当前策略：

- 不使用模拟数据。
- 不生成兜底问答。
- 不在错误时返回假成功。
- 配置缺失、模型失败、数据库失败、门禁失败都会直接暴露错误。
- 没有明确知识点时，候选抽取可以返回空。

## 14. 发布门禁规则

发布门禁不是兜底逻辑，而是发布前真实运行的检查和回归。失败会阻止发布。

### 14.1 发布前结构校验

任一失败都会阻止发布：

- 文档必须有版本号。
- 文档至少要有一条知识切片。
- 报价知识必须有生效时间和失效时间。
- 报价知识必须有结构化报价规则。
- 被识别为报价知识的条件包括：
  - 知识类型为报价知识。
  - 标题或内容包含报价、价格、收费、费用、最低收费、加急、税点、折扣、元/千字、元每千字等关键词。

### 14.2 门禁用例选择

系统会按待发布知识自动选择回归用例：

- 匹配同业务线用例。
- 高风险文档额外匹配通用风险用例。
- 匹配知识类型用例。
- 报价和高风险知识额外匹配低置信度、有效期、客户层级、跨范围等风险用例。
- 有服务范围时优先匹配对应服务范围。
- 有客户层级时优先匹配对应客户层级。
- 如果高风险文档没有匹配到用例，会兜选报价、低置信度、版本有效期三类门禁用例。

当前回归用例分组包括：

- 翻译报价。
- 翻译能力。
- 交付流程。
- 通用问答。
- 印刷销售。
- 展会销售。
- 销售话术。
- 风险防护。
- 版本有效期。
- 客户层级。
- 服务范围防串线。

### 14.3 回归判定规则

门禁会调用真实知识库检索，再按预期规则判定：

- 必须命中指定知识类型。
- 必须命中结构化报价规则。
- 应触发人工复核时必须触发。
- 不允许人工复核时不得误触发。
- 有服务范围要求时，命中证据必须匹配服务范围。
- 有业务线要求时，不得命中其他业务线。
- 有客户层级要求时，检索过滤必须带客户层级。
- 非翻译业务问题不得使用翻译报价。
- 税费和发票问题必须命中税费或发票证据。
- 流程问题必须命中流程或常见问答证据。
- 必须检查当前有效期。
- 法律范围存在证据时，不得只返回通用范围证据。
- 价格问题没有价格证据时不得返回正常结果。
- 能力问题没有能力证据时不得返回正常结果。

### 14.4 人工复核触发

以下情况会触发或倾向触发人工复核：

- 没有命中足够证据。
- 命中限制条件。
- 报价知识缺少结构化报价规则。
- 查询超出已发布知识范围。
- 命中证据置信度较低。
- 涉及价格、税费、折扣、客户层级等高风险内容但证据不足。

### 14.5 强制发布规则

系统支持强制发布，但不建议常规使用。

强制发布要求：

- 必须提供强制发布原因。
- 系统仍会记录门禁结果。
- 强制发布不会掩盖门禁失败记录。
- 高风险报价、税费、合同、承诺类知识原则上应修正后再发布。

### 14.6 门禁临时可见机制

发布门禁运行前，系统会临时让本次待发布知识参与检索，以便真实检索能看到即将发布的内容。

机制：

- 门禁前临时把待发布文档、知识切片、报价规则标记为可检索状态。
- 门禁运行真实检索和回归。
- 门禁结束后恢复原状态。
- 门禁失败不会污染线上已发布知识。
- 门禁通过后才正式发布。

## 15. 推荐操作路径

### 15.1 导入新报价单

1. 点击 `知识库管理`。
2. 点击 `导入中心`。
3. 导入类型选择 `报价规则`。
4. 点击 `下载模板`。
5. 填写标题、内容、业务线、语种、服务范围、价格、单位、币种、生效时间、失效时间。
6. 点击 `选择文件`。
7. 点击 `预览`。
8. 修正失败行。
9. 点击 `确认导入`。
10. 点击 `批次`。
11. 选择导入批次编号。
12. 点击 `提交审核`。
13. 点击 `审核同意`。
14. 点击 `发布`。
15. 如果门禁失败，按错误提示补报价规则、税费证据或有效期后重试。

### 15.2 导入流程规范

1. 点击 `知识库管理`。
2. 点击 `导入中心`。
3. 导入类型选择 `流程规范`。
4. 下载模板并填写流程标题和步骤。
5. 点击 `预览`。
6. 点击 `确认导入`。
7. 到 `批次` 依次执行 `提交审核`、`审核同意`、`发布`。
8. 到 `检索测试` 输入流程类问题验证命中。

### 15.3 从销售反馈补知识

1. 在 `企微实时智能` 中查看销售建议。
2. 如果建议需要修正，点击 `需修正`。
3. 填写修正意见。
4. 进入 `知识库管理`。
5. 点击 `知识候选`。
6. 找到候选并点击 `详情`。
7. 点击 `编辑候选` 修正文案。
8. 点击 `转为草稿知识`。
9. 依次执行 `提交审核`、`审核同意`、`发布`。

### 15.4 验证一条知识是否可用

1. 确认知识状态为 `已发布`。
2. 点击 `检索测试`。
3. 输入真实客户问题。
4. 设置业务线、知识类型、语种、服务范围。
5. 点击 `检索`。
6. 检查命中证据、报价规则、置信度和人工复核标记。
7. 如果未命中，到 `质量治理` 或 `命中日志` 生成候选并补充知识。
## 16. 服务分类口径

知识库现在只保留一个面向业务人员的服务分类，不再要求填写“子服务”。

服务下拉统一为 7 类：

- 翻译
- 印刷
- 同传
- 多媒体译制
- 展台搭建
- 礼品
- 通用

说明：

- “翻译”用于笔译、审校、资料翻译等常规翻译知识。
- “同传”用于会议口译、同传、交传等会议类语言服务，避免和普通笔译混在一起。
- “多媒体译制”用于字幕、配音、音视频译制等内容。
- “印刷”不再细分画册、手册、样本等子服务，流程和规则按同一服务处理。
- “展台搭建”用于展会、展台设计搭建、撤展等知识。
- “礼品”用于礼品定制、伴手礼、赠品等业务。
- “通用”用于公司介绍、通用流程、跨服务规则等内容。

底层数据库仍保留 `sub_service` 字段以兼容旧数据，但页面新增、编辑、导入模板和发布控制项不再使用它。
