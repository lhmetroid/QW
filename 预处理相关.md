## 一、相关代码文件

| 文件                               | 作用                                                                                 |
| -------------------------------- | ---------------------------------------------------------------------------------- |
| import_wecom_raw_direct.py       | **原始导入**：读取84MB CSV → 建表 → 批量500条流式写入DB，生成 `group_key`、`role`，所有预处理字段置NULL/pending |
| _check_db_cols.py                | 验证表结构和前5行数据（昨天确认DB可连通时用）                                                           |

## 二、字段定义

### `wecom_raw_import` 表（原始数据暂存 + 预处理）

| 字段                    | 类型           | 来源          | 枚举值 / 说明                                                         |
| --------------------- | ------------ | ----------- | ---------------------------------------------------------------- |
| `id`                  | SERIAL PK    | 自增          | —                                                                |
| `import_batch_id`     | VARCHAR(40)  | 导入时         | `wri_{文件MD5前12位}`                                                |
| `sender` / `receiver` | VARCHAR(200) | CSV原始       | `客户:wmS8sIC...` / `销售:139...`                                    |
| `group_id`            | VARCHAR(200) | CSV原始       | 群聊ID，私聊为空                                                        |
| `msg_time`            | TIMESTAMP    | CSV解析       | 消息时间                                                             |
| `content`             | TEXT         | CSV原始       | 消息内容                                                             |
| `from_id` / `to_id`   | VARCHAR(200) | CSV原始       | 企业微信内部ID                                                         |
| `role`                | VARCHAR(20)  | 导入时推断       | `customer` / `sales` / `unknown`                                 |
| `group_key`           | VARCHAR(80)  | 导入时生成       | `{batch_id}_d{N}`，N按空行分隔对话递增                                     |
| `row_index`           | INTEGER      | CSV行号       | 原文件行序                                                            |
| `stage`               | VARCHAR(50)  | **预处理赋值**   | `客户开发` / `需求确认` / `报价` / `成交` / `售后`                             |
| `section`             | VARCHAR(100) | **预处理赋值**   | `初次接触` / `关系维护` / `需求挖掘` / `方案报价` / `合同签订` / `项目执行` / `反馈回访`     |
| `business_line`       | VARCHAR(50)  | **本次新增+赋值** | `笔译` / `口译` / `印刷` / `视频译制` / `展台搭建` / `礼品定制` / `多元` / `未知`      |
| `language_pair`       | VARCHAR(100) | **本次新增+赋值** | `中英` / `中俄` / `中德` / `多语种` / `未知` / `不适用`                        |
| `process_result`      | VARCHAR(50)  | **预处理赋值**   | `入库_切片` / `不入库_纯推广` / `不入库_无信息量` / `不入库_乱码` / `待人工确认`            |
| `quality_score`       | NUMERIC(5,2) | **预处理赋值**   | 0–100，AI基于信息量+话术价值+客户反馈价值打分                                      |
| `process_note`        | TEXT         | **预处理赋值**   | 简短说明本行的关键信息或分类理由                                                 |
| `row_status`          | VARCHAR(30)  | 导入/处理更新     | `pending`（导入初始）/ `processed`（预处理完成）/ `published`（入库）/ `rejected` |
| `created_at`          | TIMESTAMP    | 自动          | 记录创建时间                                                           |

### 下游目标：`knowledge_chunk` 表（正式知识库切片）

与 `wecom_raw_import` 共享的字段含义一致，但增加了检索侧所需的字段：

| 字段                 | 说明                                                         |
| ------------------ | ---------------------------------------------------------- |
| `business_line`    | 与wecom表定义相同，用于检索过滤                                         |
| `language_pair`    | 与wecom表定义相同，用于检索过滤                                         |
| `knowledge_type`   | `capability`/`pricing`/`process`/`faq` — 知识类型，比wecom多了这层分类 |
| `chunk_type`       | `rule`/`faq`/`example`/`constraint`/`definition`           |
| `quality_score`    | 0–1小数，由KB评估worker写入（与wecom的0–100整数不同）                      |
| `useful_score`     | 检索综合有用度（占最终排序权重6%）                                         |
| `effect_score`     | 效果得分（实际使用后的反馈）                                             |
| `usable_for_reply` | 是否可用于AI生成回复                                                |
| `status`           | `draft`/`active`/`archived`                                |
