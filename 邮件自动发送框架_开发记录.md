## 二、后端(全部新增, 前缀 /api/v1/mail/autosendrules/*, 不改既有 /autosend/*)

新表(按需 CREATE IF NOT EXISTS): `mail_autosendrules_config(scope, class_rules, send_order, mailbox_mode, mail_gap_days, suite_gap_days, daily_cap, action_map, updated_at)`。scope = 销售工号 或 `__global__`。

端点:
- GET/PUT `/config` — 保存/读取分类规则、发送顺序、邮箱模式、间隔、每日上限、动作->套装映射(按 scope)。
- POST `/suggest` (需求15) — 载入该销售联系人 → 按手工分类规则归类(复用 `_autosend_rule_matches`) → LLM 按每个分类给套装排优先级(失败回退套装范围顺序) → 该联系人已发过的套装排除。只算不写。
- POST `/schedule` (需求15) — 按已确认建议+规则(发送顺序/间隔/每日上限/开始日)排期, 写入既有待发清单(status='planned'); 复用 `_autosend_existing_crm_load`/`_autosend_merge_local_plan_load` 兼容已有排期(顶满顺延、跳非工作日)。
- POST `/trigger-suggest` (需求14) — 按所选动作(跟进/丢单)的映射套装, 对候选联系人算适合套装(排除已发/已在清单)。
- POST `/trigger-confirm` (需求14) — 人工确认后写入待发清单并复用既有后台生成 worker 自动开始生成。

核心新逻辑 `_asr_schedule`(纯计算): 多套装排期, 支持两种发送顺序; 套装内第k封=起始+mail_gap*k(从起始累计), 同联系人相邻套装=前套装末封+suite_gap; 每人每日上限顶满整天顺延到下个工作日。已用独立脚本验证 contact_first / round_robin / 上限顺延+周末跳过 均正确。

## 三、口径确认(按用户 2026-07-06 答复)

1. 联系人分类 = 手工逐条规则, 不限个数; 顺序=优先级; 命中第一条即归类。条件维度: 累计销售合同额`<`/`>`、距上次合作天数`<`/`>`、业务线 合作过/没合作过。
2. 发送顺序两方案都做、独立可选; 兼容已有排期。
3. 已发套装排除口径 = 该联系人是否发过该套装(有任一已发封即整套装排除)。
4. 跟进/丢单先下拉选择; 动作事件源后续对接外部系统。
5. 排期写入既有待发清单 `mail_autosend_plan_item`。

## 四点六、需求17/18/19 已完成(2026-07-07 追加)

- 需求19 公共邮箱账号配置(自动发送节点 模式③): 新表 `mail_public_mailbox(id,email,display_name,daily_limit,enabled)`。端点 GET `/api/v1/mail/mailboxes`、POST `/api/v1/mail/public-mailbox`(增改)、DELETE `/api/v1/mail/public-mailbox?id=`。前端可增/改/删公共邮箱及每天上限。个人邮箱由各销售自身账号决定, 此处只管理公共邮箱。
- 需求17 公共邮箱使用规则: 新表 `mail_public_mailbox_policy(strategy, ratio_json)`。策略 first_come(先到先得, 全部公共容量全销售共享)/ ratio(每销售按权重分配公共当日容量)。端点 PUT `/api/v1/mail/public-mailbox-policy`。已接入排期 `_asr_schedule`:
  - personal_only=只个人; public_only=只公共; personal_first=个人当日满则转公共(需求17"销售当天满用公共邮箱")。
  - 各公共邮箱有独立每日上限; ratio 下每销售当日公共配额=权重占比 x 公共总容量; 皆满则顺延工作日。
  - 每封记录发件箱: 待发清单表新增列 `asr_sender_kind`(personal/public) + `asr_sender_email`(ALTER IF NOT EXISTS)。schedule 返回 public_count。
- 需求18 哪封邮件回复最多(模式④): 只读端点 GET `/api/v1/mail/reply-ranking?start=&end=&staff_id=`, 由 `mail_ai_reply_analysis` JOIN `mail_customer_suite_send_plan(plan_id)` 按 套装+第几封 聚合回信数与有价值回信数, 回信多者排前。前端表格展示, 销售范围跟随顶部选择。
- 修复: 公共邮箱 email 不能过 `sanitize_text`(该函数做PII脱敏会抹掉 @ 导致校验失败), 改为直接 strip。
