# 邮件去重与入库评估

## 结论

- `docs/邮件.csv` 不适合整批直接作为正式知识发布。
- 原因不是文件不能解析，而是内容大多是项目级报价、发票、PO、交付通知，噪声高且变量过多。
- 其中有一部分高频表达可以抽象成 `邮件模板 / 流程规则 / 报价限制条件 / FAQ常见问答`，已整理为新的入库准备文件。

## 去重结果

- 原始非空邮件数：`935`
- 完全相同重复行：`0`
- 归一化后唯一模板骨架数：`915`
- 可归并的重复模板组数：`10`

说明：这里的“归一化去重”会把客户称呼、金额、日期、页数、字数等变量替换成占位符，再识别是否属于同一模板骨架。

## 高频场景统计

- 报价确认/启动：`169`
- 报价明细：`169`
- 申请PO：`97`
- 同传Book：`10`
- 发票付款：`99`
- 简单附件通知：`226`

## 价值判断

- 高价值：可复用的报价确认、申请PO、同传Book、发票付款、交付盖章、报价更新等邮件表达。
- 中价值：流程或限制条件，例如“未确认则顺延交期”“收到PO后开始工作”“打样确认后启动生产”。
- 低价值：纯附件通知、单次项目金额、文件名、具体交期日期、单个客户的事务跟进。

## 入库建议

- 正式知识：只导入已经抽象成标准模板或规则的内容。
- 候选池：如后续要做更深抽取，可把剩余原始邮件走“邮件整理表抽取 -> 候选池”链路，不建议直接发布。
- 已生成准备入库文件：`docs/邮件_知识库准备.csv`，共 `18` 条。
- 已生成归一去重结果：`docs/邮件_归一去重结果.csv`。
