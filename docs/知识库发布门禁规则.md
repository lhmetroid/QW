# 知识库发布门禁规则

本文档记录当前代码里的发布门禁规则。门禁不是兜底逻辑；它在知识发布前真实运行检索回归，失败时直接阻止发布。

## 1. 发布前结构校验

发布入口会先执行文档级校验，任一失败都会直接返回错误：

- 文档必须有 `version_no`。
- 文档至少要有 1 条 `knowledge_chunk`。
- 只要 chunk 被识别为报价知识，就必须满足报价发布要求。
- 报价知识识别条件：`knowledge_type=pricing`，或标题/内容包含 `报价`、`价格`、`收费`、`费用`、`最低收费`、`加急`、`税点`、`折扣`、`元/千字`、`元每千字`。
- 报价文档必须有 `effective_from` 和 `effective_to`。
- 报价 chunk 必须有结构化 `pricing_rule`。

## 2. 发布门禁用例选择规则

门禁会按本次待发布文档和 chunk 自动选择回归用例：

- 匹配同业务线的用例，例如 `translation`、`printing`、`exhibition`、`general`。
- 高风险文档会额外匹配 `general` 业务线用例。
- 匹配知识类型，例如 `pricing`、`capability`、`process`、`faq`。
- 高风险或报价知识会额外匹配低置信度、版本有效期、客户层级、跨范围等风险用例。
- 如果 chunk 有 `service_scope`，会优先匹配对应服务范围，跨范围和能力类用例除外。
- 如果 chunk 有 `customer_tier`，会优先匹配对应客户层级，客户层级风险用例除外。
- 如果高风险文档没有匹配到任何用例，会兜选 `pricing`、`low_confidence`、`versioning` 三类用例。
- 当前回归集共 100 条，分组包括 `translation_pricing`、`translation_capability`、`delivery_process`、`general_faq`、`printing_sales`、`exhibition_sales`、`sales_script`、`risk_guard`、`version_guard`、`crm_tier`、`scope_guard`。

## 3. 回归结果判定规则

每条回归用例会调用真实 `IntentEngine.retrieve_knowledge_v2`，再按 `expected` 字段判定：

- `must_hit_knowledge_type`：命中结果里必须包含指定 `knowledge_type`。
- `must_hit_pricing_rule`：命中结果里必须包含结构化 `pricing_rule`。
- `manual_review_required=true`：检索结果必须触发人工复核。
- `manual_review_allowed=false`：检索结果不得触发人工复核。
- `service_scope_filter`：有命中时，至少一条命中必须是指定 `service_scope`。
- `business_line_filter`：有命中时，所有命中必须属于指定 `business_line`。
- `customer_tier_filter`：检索过滤条件必须实际带上指定 `customer_tier`。
- `must_not_use_translation_price`：非翻译业务查询不得命中翻译报价知识。
- `must_hit_tax_or_invoice`：命中证据必须包含税、发票、`invoice` 或 `tax` 相关内容。
- `should_return_process_or_faq`：必须命中 `process` 或 `faq` 证据。
- `must_check_effective_time`：检索过滤必须带当前有效期过滤，`filters_used.effective_time=now`。
- `must_not_use_general_if_legal_exists`：请求法律范围时，如果存在法律范围证据，不得只返回通用范围证据。
- `should_include_rules_before_expression`：建议先命中规则类证据；当前只作为 warning，不直接阻断。
- `must_not_fabricate_price`：价格类问题不能在无价格证据时返回 `ok`。
- `must_not_fabricate_capability`：能力类问题不能在无能力证据时返回 `ok`。

## 5. 已修复的门禁流程缺陷

原逻辑在发布前运行门禁，但检索只查 `active` 知识，导致本次待发布的 `review` 知识无法参与门禁检索。

现已调整为：

- 门禁运行前临时把本次待发布文档、chunk、pricing_rule 标记为 `active` 并提交，使真实检索能看到待发布内容。
- 门禁运行结束后恢复原状态。
- 门禁失败时不污染线上 active 集合。
- 门禁通过后再由原发布流程正式设置 `active/approved`。
