# 生产部署与配置清单

本文档用于交付 `QW` 项目的生产部署、配置核对和故障排查。当前前端运行状态页对应后端接口：

* `GET /api/health/ready`
* `GET /api/system/config_check`

当前公网对外地址建议固定为：

* `EXTERNAL_API_BASE_URL=https://api.speedasia.net`
* `SALES_KB_API_BASE_URL=https://knowledgebase.speedasia.net`

## 1. 基础环境

建议生产机至少满足：

* Windows Server 或稳定的 Windows 10/11 专机
* Python 3.11+
* PostgreSQL 14+
* 可访问 Ollama / 内网 LLM 网关
* 如启用客户画像或 CRM 融合，可访问 CRM SQL Server
* 具备 HTTPS 反向代理能力

目录建议：

```text
D:\items\QW\
  backend\
  frontend\
  docs\
  logs\
```

## 2. PostgreSQL

需要确认以下配置：

* `DB_HOST`
* `DB_PORT`
* `DB_NAME`
* `DB_USER`
* `DB_PASSWORD`

检查项：

* 数据库实例已启动。
* 目标库已创建。
* 当前主机允许访问 PostgreSQL。
* 账号具备读写业务表权限。

建议验证：

```powershell
psql -h <DB_HOST> -p <DB_PORT> -U <DB_USER> -d <DB_NAME> -c "select 1;"
```

## 3. Ollama / Embedding

需要确认以下配置：

* `EMBEDDING_PROVIDER=ollama`
* `EMBEDDING_API_URL`
* `EMBEDDING_MODEL`
* `EMBEDDING_TIMEOUT_SECONDS`

检查项：

* Ollama 服务已启动。
* `/api/tags` 可访问。
* 指定 embedding 模型已拉取完成。
* 代理不会错误转发内网请求。

建议验证：

```powershell
curl http://<OLLAMA_HOST>:11434/api/tags
```

## 4. LLM 网关

系统当前区分两套模型配置：

* `LLM1_API_URL` / `LLM1_API_KEY` / `LLM1_MODEL`
* `LLM2_API_URL` / `LLM2_API_KEY` / `LLM2_MODEL`

检查项：

* URL 可达。
* API Key 非占位符。
* 模型名与网关实际可用模型一致。
* 如走代理，代理规则允许访问。

如果使用 Dify 类型 `app-` key：

* 确认对应应用已发布。
* `/info` 能返回正常响应。

如果使用 OpenAI 兼容网关：

* `/models` 可访问。
* `/chat/completions` 可调用。

## 4.1 知识库2直连 API

知识库2当前不再走旧内网 IP，生产环境统一通过反向代理后的 HTTPS 域名访问：

* `SALES_KB_API_BASE_URL=https://knowledgebase.speedasia.net`
* 实际调用接口：`POST /kb-units/qa-retrieve`

检查项：

* 必须使用 `https`。
* 不应携带端口号。
* 不应再配置为 `192.168.31.124` 一类旧内网地址。
* 服务器本机需要能直接访问该域名。

建议验证：

```powershell
$body = @{ question = '付款' } | ConvertTo-Json
Invoke-WebRequest `
  -Uri 'https://knowledgebase.speedasia.net/kb-units/qa-retrieve' `
  -Method POST `
  -ContentType 'application/json' `
  -Body $body
```

## 5. CRM 数据库

CRM 是只读辅助链路，不是知识库核心依赖。知识库导入、检索、审核、发布只依赖 `AISaleData` PostgreSQL；CRM 仅在客户画像、企微侧边栏辅助和 LLM 上下文增强时读取旧库。

启用 CRM 辅助时需要确认以下配置：

* `CRM_DBHost`
* `CRM_DBPort`
* `CRM_DBName`
* `CRM_DBUserId`
* `CRM_DBPassword`

检查项：

* SQL Server 网络可达。
* ODBC Driver 已安装。
* 账号可执行基础查询。

建议验证：

```powershell
sqlcmd -S <CRM_DBHost>,<CRM_DBPort> -d <CRM_DBName> -U <CRM_DBUserId> -P <CRM_DBPassword> -Q "SELECT 1"
```

## 6. 企业微信应用

企业微信应用是外部接入链路，不是知识库核心依赖。企微回调暂缓时，`TOKEN`、`ENCODING_AES_KEY` 可为空；不发送应用卡片/消息时，`AGENT_ID` 可为空。

启用企微应用接入时需要确认以下配置：

* `CORP_ID`
* `CORP_SECRET`
* `AGENT_ID`
* `TOKEN`
* `ENCODING_AES_KEY`

检查项：

* 应用 Secret 未过期。
* 回调地址与加解密参数一致。
* 企业微信应用可正常发消息。

## 7. 会话存档 SDK

会话存档 SDK 只影响“同步今日真实记录”和从企微历史消息读取，不影响知识库后台读取、导入、检索和发布。

启用会话存档时需要确认以下文件与配置：

* `backend/WeWorkFinanceSdk.dll`
* `PRIVATE_KEY_PATH`
* `CHATDATA_SECRET`

检查项：

* DLL 文件存在。
* 私钥文件存在。
* 会话存档密钥正确。

## 8. HTTP 代理与内网直连

如果生产机配置了系统代理，重点检查：

* Ollama 内网地址是否绕过代理。
* LLM 内网网关是否绕过代理。
* `HTTP_TRUST_ENV` 是否符合当前部署策略。

建议：

* 内网服务优先直连。
* 仅外网模型或外部 API 走代理。
* 保留代理变更记录，避免后续排障困难。

## 9. HTTPS 与反向代理

建议通过 Nginx 或 IIS 反向代理对外提供 HTTPS。

如果当前采用 `IIS` 作为公网 HTTPS 入口，建议后端本机直接运行明文 HTTP，不要再让 FastAPI 自己启本地 HTTPS。推荐组合如下：

* `EXTERNAL_API_BASE_URL=https://api.speedasia.net`
* `ENABLE_LOCAL_HTTPS=false`
* `CLOUDFLARED_TARGET_URL=http://localhost:8071`
* IIS / ARR 反代目标：`http://127.0.0.1:8071`

至少需要：

* TLS 证书
* 反向代理到 FastAPI 服务端口
* 大请求体支持，避免 Excel 上传失败
* 适当的超时设置，覆盖导入、回归、知识抽取场景

建议关注：

* `client_max_body_size`
* upstream/read timeout
* 访问日志与错误日志分离

当前建议的公网映射：

* Base URL: `https://api.speedasia.net`
* Swagger: `https://api.speedasia.net/docs`
* Frontend: `https://api.speedasia.net/static`
* 企微回调: `https://api.speedasia.net/cb/qywx`

## 10. 服务守护

建议将后端作为常驻服务运行，避免人工手动启动。

可选方案：

* NSSM 注册为 Windows Service
* Task Scheduler 开机自启
* IIS + FastCGI / 反向代理

建议：

* 服务启动账号固定
* 日志目录可写
* 重启策略开启
* 异常退出自动拉起

## 11. 发布前检查清单

上线前至少确认：

1. `GET /api/health/ready` 返回 `status=ok` 或明确可接受的 `degraded` 原因。
2. `GET /api/system/config_check` 中 PostgreSQL、Embedding、LLM 必须正常；CRM、企微应用、会话存档 SDK 按是否启用对应功能核对。
3. `GET /api/system/config_check` 中 `sales_kb` 必须显示为 `ok`，且基址为 `https://knowledgebase.speedasia.net`。
4. 前端知识库运行状态页可以看到配置诊断结果。
5. 关键环境变量不再使用占位符值。
6. HTTPS、日志目录、服务守护均已就绪。
7. 若当前只允许 API 输出单版本，确认：
   * `API_REPLY_SINGLE_MODEL_SINGLE_STYLE=true`
   * `API_REPLY_ENABLE_SCORING=false`

### 12.3 知识库2连不上

表现：

* 运行状态页中 `知识库2 API` 显示异常
* `GET /api/health/ready` 返回 `sales_kb.status=error`
* 第 05 步知识库2显示 `skipped_unavailable`

处理：

* 核对 `SALES_KB_API_BASE_URL` 是否为 `https://knowledgebase.speedasia.net`
* 确认没有端口号，没有旧内网 IP
* 在服务器本机执行上面的 `Invoke-WebRequest` 探测
* 若域名可通但接口异常，检查反向代理目标服务和 TLS 证书

## 12. 常见故障排查

### 12.1 Ollama 不可用

表现：

* 配置诊断中 `embedding.status=error`
* 健康检查返回 `embedding.error`

处理：

* 确认 Ollama 进程和端口。
* 检查模型是否已拉取。
* 检查代理是否错误接管内网地址。

### 12.2 LLM 调用失败

表现：

* `llm1` 或 `llm2` 探测失败
* 侧边栏无法生成分析或话术

处理：

* 核对 URL、API Key、模型名。
* 检查网关是否支持 `/models` 或 `/info`。
* 检查超时和代理。

### 12.3 CRM 不可用

表现：

* `crm.status=error`
* 客户画像为空或报错

处理：

* 检查 SQL Server 地址、端口、ODBC Driver 和账号权限。

### 12.4 企微会话存档失败

表现：

* `archive_sdk.status=error`

处理：

* 检查 SDK DLL、私钥路径、会话存档密钥配置。
