# 功能清单与使用说明

本文档按当前前端页面整理。系统当前不提供登录页；开发期直接访问前台地址即可使用。生产上线前如需登录、角色和按钮权限，需要另行接入鉴权。

## 1. 启动与访问

1. 复制 `.env.example` 为 `.env`。知识库核心链路必须填写真实知识库数据库、向量模型服务和大模型配置；客户画像库、企业微信回调、企业微信应用编号和会话存档配置为外部增强功能，未配置时只影响对应功能，不影响知识库导入、检索、审核和发布。
2. 双击项目根目录 `一键启动后台.bat`，或执行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\start_backend.ps1
```

3. 浏览器打开：

```text
http://127.0.0.1:8000/
```

4. 当前无账号密码登录流程；打开页面即进入系统。顶部有两个并排大节点：`企微实时智能` 和 `知识库管理`。
5. 点击 `知识库管理` 后进入知识库总页面，不再弹出遮罩窗口。知识库内部通过页签切换：`列表`、`待审核`、`知识候选`、`导入中心`、`批次`、`报价规则`、`检索测试`、`质量治理`、`回归测试`、`运行状态`、`命中日志`。
6. 可直接访问 `http://127.0.0.1:8000/#kb-import` 打开导入中心。

说明：企微会话存档同步是可选功能，默认 `ENABLE_ARCHIVE_POLLING=false`，后台启动不会自动读取会话存档。当前私钥路径会按项目根目录解析，例如 `PRIVATE_KEY_PATH=backend/private_key.pem` 对应 `D:\items\QW\backend\private_key.pem`。只有在补齐 `WeWorkFinanceSdk.dll`、`PRIVATE_KEY_PATH`、`CHATDATA_SECRET` 并显式开启 `ENABLE_ARCHIVE_POLLING=true` 后，才会启动自动轮询。

## 2. 顶层节点

- `企微实时智能`：查看真实企微会话、同步会话存档、提取销售意图、读取客户画像、调用知识库证据生成建议。
- `知识库管理`：导入基础数据、查看知识、维护报价规则、审核发布、检索测试、质量治理、回归测试和查看命中日志。

## 3. 企微实时智能

- `同步今日真实记录`：调用企微会话存档组件，同步当天真实消息；需先配置组件、私钥和会话存档密钥，未配置时会直接报明确错误，不影响知识库后台启动。
- `引擎配置`：查看和编辑智能引擎提示词与快速扫描规则。
- 会话列表：只展示 `message_logs.is_mock=false` 的真实会话。
- `刷新分析`：重新读取当前会话摘要、客户画像、知识证据和建议。
- `立刻调用智能模型提取特征`：使用第一阶段大模型对当前真实会话生成结构化摘要。
- `立刻调用知识库进行话术辅助生成`：使用知识库证据、客户画像和第二阶段大模型生成销售建议。
- `复制 / 有用 / 需修正`：复制建议、写入反馈，或把纠错反馈转为知识候选。

## 4. 知识库管理

- `列表`：查看知识文档，支持状态、知识类型、业务线筛选。
- `新增知识`：手工写入草稿知识，保存时会真实调用向量模型。
- `详情`：查看文档、知识切片、报价规则、版本历史。
- `编辑文档 / 编辑知识切片`：修改文档或切片；修改知识切片会重建向量。
- `补报价规则 / 编辑 / 停用`：维护结构化报价证据。
- `提交审核 / 发布 / 归档 / 驳回`：治理知识状态；发布会执行发布门禁。
- `恢复到此版本`：按历史快照恢复文档、知识切片和报价规则，并重新进入审核。

### 4.1 知识类型区分

- `能力知识`：回答“能不能做、支持哪些语种、覆盖哪些服务”。示例：中译德可做、支持英日法德俄韩等。
- `报价知识`：回答价格、计价单位、最低收费、税费、折扣等。报价知识属于高风险知识，必须尽量补结构化报价规则，发布门禁会重点检查。
- `流程知识`：回答“怎么做、步骤是什么、交付怎么走”。示例：下单流程、付款流程、审核流程、交付流程。
- `常见问答`：承载普通解释、问答、话术和案例摘要。常见问答不能替代报价规则，也不能承诺未结构化的价格。

### 4.2 切片类型区分

- `规则`：可执行、可判断的标准条款，例如报价规则、付款规则、服务边界。规则优先级高，适合参与正式检索判断。
- `问答`：一问一答式知识，用于稳定、低风险的标准回答。
- `案例`：历史成交或沟通案例，用于参考话术和场景经验，不应直接当作当前价格或承诺。
- `话术模板`：销售可复用表达模板。模板必须基于已发布证据输出，不能编造价格、交期或能力。
- `限制条件`：不能做、必须人工确认、需要补充信息等边界规则。命中后通常要求人工复核。
- `定义`：解释术语、字段、业务概念。定义用于提高检索理解，不承载业务承诺。

### 4.3 状态规则

- `草稿`：已写入数据库但未审核，不参与正式检索。
- `审核中`：等待发布前检查和人工确认。
- `已发布`：正式参与检索和销售建议生成。
- `已归档`：停止参与检索，保留历史记录。
- `已驳回`：审核不通过，不能发布。

## 5. 审核、候选与导入

- `待审核`：批量提交审核、发布或归档。
- `知识候选`：查看销售反馈、会话抽取、邮件抽取生成的候选知识。
- `编辑候选`：修正候选字段和内容。
- `转为草稿知识`：候选转入知识库草稿，后续走审核发布。
- `导入中心`：选择常见问答、报价规则、流程规范、能力清单、历史案例、销售话术、邮件整理类型。
- `下载模板`：下载对应表格模板。
- `预览`：只校验表格，不写库。
- `确认导入`：写入真实知识库草稿并生成导入批次编号。
- `会话抽取 / 历史案例抽取 / 邮件整理表抽取`：使用真实大模型从真实内容中抽取候选；没有明确知识点时返回空，不生成兜底问答。
- `批次`：按批次提交审核、发布、归档、回滚。

## 6. 检索、质量与回归

- `报价规则`：集中查看结构化价格规则。
- `检索测试`：输入客户问题，选择业务线、知识类型、语种、服务范围，查看命中知识切片、报价规则、证据分数和过滤条件。
- `质量治理`：发现缺失报价规则、重复/冲突规则、过期文档或规则。
- `详情 / 文档 / 知识切片 / 规则 / 补规则 / 归档重复 / 延长期限 / 停用规则`：从质量问题直接跳转修复。
- `回归测试`：按分组、类别、风险、业务线筛选商业验收集并异步执行。
- `命中日志`：查看真实检索日志，把需要修正的问题转为候选知识。

## 7. 运行状态

- `健康检查`：检查知识库数据库和向量模型服务。
- `检索性能`：查看索引、检索策略、知识数量。
- `配置诊断`：检查知识库数据库、向量模型服务、第一阶段大模型、第二阶段大模型、客户画像库、企微应用、会话存档组件。

## 8. 真实端到端联调

知识库核心环境变量仍是占位值时，一键启动会直接停止并提示补齐 `.env`。配置完成后可执行：

```powershell
cd backend
python .\real_e2e_check.py --excel "..\docs\业务知识、业务流程、话术、案例.xlsx" --import-type faq --query "英译法普通商务资料多少钱" --query-features "{\"business_line\":\"translation\"}"
```

该脚本会依次检查：

- 知识库数据库连接。
- ORM 与真实数据库字段对齐。
- 向量模型可用。
- 表格导入、提交审核、发布门禁、发布。
- 知识检索。
- 第二阶段大模型基于证据生成建议。

如需同时联调只读 CRM 客户画像，再追加：

```powershell
--include-crm --external-userid "<真实外部联系人ID>"
```

客户画像使用旧库 `1e79f6355.zicp.vip:3301/DBCenterNew2`，只用于客户画像和大模型辅助上下文读取；知识库新数据只写入 `AISaleData`。
开启 `--include-crm` 后，客户画像查询失败会直接抛错，不返回兜底画像。

任一环节失败会直接抛错，不使用模拟数据或兜底答案。
