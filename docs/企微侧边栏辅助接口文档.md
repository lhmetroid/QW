# 企微侧边栏辅助接口文档

## 1. 接口概述

该接口供企微右侧边栏同步拉取当前会话的结构化摘要、CRM 画像、知识证据与二阶段回复建议。

- 接口地址：`POST /api/wecom/sidebar_assist`
- Content-Type：`application/json`

## 2. 请求参数

```json
{
  "external_userid": "wmxxxx",
  "userid": "WangHuiYing",
  "force_refresh": false,
  "sync_archive_before_read": true
}
```

字段说明：

| 字段名 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| `external_userid` | string | 是 | 企微外部联系人 ID |
| `userid` | string | 是 | 销售企微账号 |
| `force_refresh` | boolean | 否 | 是否强制重跑 LLM 分析 |
| `sync_archive_before_read` | boolean | 否 | 是否先同步企微会话存档 |

## 3. 关键响应字段

二阶段回复建议已从单字段改为两个独立字段，且字段值本身不再包含 `【企微回复参考】`、`【跟进思路说明】` 这两个标签。

| 字段名 | 类型 | 说明 |
|---|---|---|
| `reply_reference` | string\|null | 可直接复制发送给客户的企微回复 |
| `followup_rationale` | string\|null | 给销售看的跟进思路说明 |
| `reply_reference_compare` | string\|null | 对比模型生成的可发送回复 |
| `followup_rationale_compare` | string\|null | 对比模型生成的跟进说明 |
| `reply_style_results_v2` | array | 多风格、多模型候选结果；每个候选同样拆分为 `reply_reference` 与 `followup_rationale` |

说明：

- `sales_advice_v2`、`sales_advice_compare_v2` 仍作为后端内部持久化字段保留。
- 对外接口返回时，不再直接回传这两个组合文本字段。

## 5. 兼容性说明

- 旧前端若仍依赖 `sales_advice_v2` 或 `sales_advice_compare_v2`，需要同步改造。
- 若读取 `reply_style_results_v2`，请改为消费候选对象内的 `reply_reference` 与 `followup_rationale`。
